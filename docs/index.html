MARK · Audit

Audit system

MARK's Audit mode pulls a half's collected data from Tag Once's Apollo cache via the bridge and scores the collector's work based on the QA reviewer's amendments.

How it works

The bridge is a JS script injected into Tag Once's app.asar when the user clicks "Embed Bridge". It reads the Apollo cache — base events, amendments, refinements, lineup players — and sends everything via a localhost WebSocket to MARK. MARK computes the score, renders the errors table, and saves to Firestore + Drive.

Three roles per half

RoleDetection (diagnostics.work)Example
Base Collectorbase > 3A-088 Alaa Wagih
Specialist Collectorrefinement > 3 AND base === 0A-2454 Mohamed Adel (players + location)
Revieweramendment > 0 AND base === 0 AND refinement === 0A-1437 Omar Salama

Multiple reviewers per half are supported. All people meeting the reviewer rule are included in reviewerIds.

MARK · Scoring

The correct scoring rule

⚠️
All audit scores before v7.5.20 are wrong The specialist collector was misidentified as a reviewer, inflating errors by ~200 per half. Pre-v7.5.20 sessions show deflated quality scores (~15pp lower than correct).

Confirmed by console testing on match 1436691 (Moss vs Sandnes, 1st half, 1,384 base events):

Old MARK (wrong)
77%
313 error keys · includes specialist
Correct (v7.5.20+)
92%
110 error keys · reviewer only

The rule

Error = amendment authored by the reviewer ONLY

Score = (total_base_events - unique_error_event_keys) / total_base_events

NOT errors:
  - Cross-collector corrections (A ↔ B)
  - Self-corrections (collector fixing own work)
  - System amendments (pairKey + no author)
  - System amendments (location desired:null + no author)

MARK · Bridge

The bridge

A JS script injected into Tag Once's app.asar (the app's HTML via patching). Runs inside Tag Once, reads its Apollo cache, and sends results over a localhost WebSocket (port 9001) to MARK.

🚨
Critical rule: BRIDGE_VERSION and ASAR_MARKER must match the app version on every bump Both were stuck at '7.5.4' for 20 builds (v7.5.5–v7.5.24). Every "Embed Bridge" click silently did nothing. All bridge improvements were silently never delivered to users.

What the bridge sends

FieldContents
baseEventsAll base events for the half
amendmentsAll amendments for the half
refinementsNew v7.5.23: {key_type: payload} for all reviewed events
lineupPlayersHome + away players with jersey, teamId, teamName
identitieslegacyId → {hrcode, name, email} from EventHistory.authorInfo
diagnostics.workPer-author: base, refinement, amendment, views counts
moduleScoresPre-computed per-module quality scores

Why refinements must come from the bridge

MARK previously tried to read Before values (extras, location, players) from the Apollo cache in the MARK webview. This only works if Tag Once has the same match open at the exact time of rendering — which fails for restored sessions. The bridge always has the right match open at audit time, so it's the authoritative source.

Embed procedure

1. Close Tag Once completely
2. Open MARK (new version)
3. Click ⚡ Embed Bridge
   → Collection app console must show:
      [MARK] bridge starting (v7.5.28 — localhost WebSocket)
4. Open Tag Once
5. Click Get Audit Results

Architecture

Key decisions

Error rule — reviewer only (2026-07-11)

Confirmed from live console testing. The specialist collector (players+location, zero base events) was misidentified as a reviewer via telemetry. Cross-collector corrections and self-corrections are not errors. Only the QA reviewer's amendments count. Multiple reviewers per half supported. Error attribution: counts against the person who did that module's work (specialist gets the error, not the base collector).

A/B/C review groups — deferred indefinitely

Tag Once's A/B/C categories use internal StatsBomb filtering logic (e.g. "Pressures Before Shots" = only pressures within N seconds of a shot). This cannot be reproduced from event names alone, and is not exposed in the Apollo cache or the JS bundle. MARK keeps its own Base/Pressure/Extras/Players/Location/FreezeFrame grouping.

Refinements from bridge, not Apollo cache

MARK's AmendmentsTable previously read Before values from the Apollo cache in the MARK webview. This fails for restored sessions (cache may have a different match). The bridge always has the right match open at audit time — so refinements now travel in the bridge's WebSocket message directly.

Open in Drive — rundll32, not anchor tags

<a href target="_blank"> is a no-op in a Tauri WebView (no browser context). All external URLs must use invoke('open_file', { path: url }) which calls rundll32 url.dll,FileProtocolHandler on Windows.

Sheet visual identity — under investigation

Sheets API batchUpdate sends 200 OK but formatting is not applied. Likely cause: service account token may not include https://www.googleapis.com/auth/spreadsheets scope, or Shared Drive restricts programmatic formatting. Next step: log full response body, verify service account scopes, test on personal Drive file.
